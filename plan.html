🔑
Clave del sistema: Claude con Tool Use
Claude recibe el mensaje del paciente + historial de conversación y puede llamar funciones: buscar_disponibilidad(especialidad, fecha), crear_cita(rut, profesional, hora), cancelar_cita(rut, cita_id). Esto permite conversación 100% natural sin necesidad de menús ni botones.
📋 Fases de Implementación
Fase 1
💬 WhatsApp conectado
Conectar el número de WhatsApp Business al sistema y recibir/enviar mensajes básicos. Permite validar el canal antes de agregar lógica.
  • Crear cuenta Meta Business Manager
  • Solicitar acceso WhatsApp Business API
  • Configurar número de teléfono verificado
  • Levantar servidor FastAPI con endpoint /webhook
  • Verificar webhook con token (Meta lo exige)
  • Enviar y recibir un "Hola" de prueba
  • Deploy en servidor con HTTPS (ngrok en dev)
~2 semanas
Fase 2
🧠 Chatbot con Claude
Integrar Claude AI para responder preguntas frecuentes del centro médico con lenguaje natural. Sin agenda aún.
  • Integrar API de Anthropic (Claude)
  • Diseñar prompt del sistema (contexto CMC)
  • Persistir historial en PostgreSQL
  • Gestionar sesiones por número de teléfono
  • Responder: horarios, precios, especialidades
  • Limitar a mensajes de texto (no audio/imágenes aún)
  • Panel admin básico: ver conversaciones
~2 semanas
Fase 3
🔗 Integración Medilink
Conectar el bot con Medilink para consultar disponibilidad real. Es la fase más compleja, depende de si Medilink tiene API.
  • Investigar si Medilink tiene API pública/privada
  • Alternativa: scraping autenticado del portal
  • Función: buscar_disponibilidad(especialidad, fecha)
  • Función: buscar_paciente(rut)
  • Caché de disponibilidad (Redis, 5 min TTL)
  • Manejo de errores y fallback manual
crítico — depende Medilink
Fase 4
📅 Agendamiento Natural
El paciente puede agendar, cancelar y reprogramar horas hablando naturalmente. Claude usa "tools" para ejecutar acciones en Medilink.
  • Definir tools: crear_cita, cancelar_cita, listar_citas
  • Flujo de confirmación antes de ejecutar
  • Extracción de RUT chileno desde texto
  • Validación de RUT (módulo 11)
  • Recordatorio automático 24h antes (cron)
  • Manejo de ambigüedad: "mañana", "la próxima semana"
  • Pruebas con pacientes reales (beta)
~3 semanas
Fase 5
📊 Dashboard de Monitoreo
Panel para que el equipo del centro médico vea conversaciones, métricas y pueda intervenir cuando el bot no sabe qué responder.
  • Conversaciones en tiempo real
  • Métricas: agendas creadas, tasa de éxito
  • Intervención manual: el admin responde directamente
  • Alertas: bot atascado > 3 turnos sin resolver
  • Reportes: horas más consultadas, especialidades
  • Blacklist/whitelist de números
opcional / mejora continua

🔗 Integración con Medilink
🔍
Opciones de conexión
De mejor a peor opción
1. API oficial de Medilink
Medilink puede tener API REST para integraciones. Hay que contactar a su soporte técnico y solicitar credenciales de integración. Es la opción más robusta y estable.
Mejor opción
2. Automatización web (Playwright)
Si no hay API, se puede automatizar el navegador: login → navegación → extracción de datos. Funciona pero es frágil ante cambios de UI de Medilink.
Plan B
3. Integración manual / híbrida
El bot recopila los datos del paciente y envía un formulario al equipo del centro. Un humano confirma en Medilink. Sin automatización pero funcional desde el día 1.
Inicio rápido
📡
Funciones necesarias
Lo que el bot necesita hacer en Medilink
buscar_disponibilidad()
→ Especialidad + rango de fechas
← Lista de slots disponibles con hora y profesional
buscar_paciente(rut)
→ RUT del paciente
← Nombre, historial de citas, datos de contacto
crear_cita()
→ RUT + profesional_id + datetime
← ID de confirmación + detalles de la cita
cancelar_cita(cita_id)
→ ID de la cita
← Confirmación de cancelación

⚠️ Riesgos y Mitigaciones
Alto
Medilink sin API pública
Medilink puede no tener API REST y el scraping es frágil. Un cambio de UI rompe la integración.
Iniciar con modo híbrido (bot recopila datos, humano agenda en Medilink). Desarrollar scraping como mejora posterior.
Alto
Verificación Meta Business
Meta puede tardar semanas en verificar la cuenta empresarial. Sin esto no hay WhatsApp API.
Usar Twilio Sandbox para desarrollo. Iniciar proceso de verificación Meta desde el día 1 del proyecto.
Medio
Bot malinterpreta fecha/hora
"La próxima semana" o "mañana" pueden ser ambiguos. Una cita mal creada genera mala experiencia.
Siempre confirmar con fecha exacta antes de crear. "¿Confirmas para el jueves 28 a las 09:30?" Nunca crear sin confirmación explícita.
Medio
RUT incorrecto o inválido
Pacientes pueden escribir su RUT con errores o dar el RUT de otro.
Validar dígito verificador (módulo 11) antes de buscar en Medilink. Pedir confirmación del nombre completo.
Medio
Bot responde algo incorrecto
Claude puede alucinar precios o disponibilidad si el prompt no está bien definido.
Solo informar precios desde datos hardcodeados en el prompt. Nunca inventar disponibilidad; siempre consultar Medilink en tiempo real.
Bajo
Spam o abuso del bot
Alguien podría intentar crear o cancelar citas masivamente.
Rate limiting por número de teléfono. Blacklist. Confirmación por código SMS para cancelaciones.

Cuenta Meta Business + verificación. Repositorio Git. Servidor con HTTPS. Endpoint /webhook funcionando. Primer mensaje enviado y recibido.
Semana 3
🟢 Contactar Medilink
Email/llamada al soporte de Medilink para consultar API. Mientras: diseñar las funciones mock con datos de prueba.
Semana 4–5
🔵 Bot conversacional básico
Claude responde preguntas frecuentes. Historial en PostgreSQL. Gestión de sesiones. Responde sobre horarios, precios, cómo llegar.
Semana 6
🔵 Modo híbrido de agendamiento
Bot recopila datos del paciente (nombre, RUT, especialidad, horario deseado) y envía por WhatsApp/email a secretaría para confirmar en Medilink.
Semana 7–9
🟣 Integración Medilink
API o scraping de Medilink. Disponibilidad real. Bot consulta horarios en tiempo real. Beta con 5–10 pacientes conocidos del centro.
Semana 10–12
🟠 Agendamiento automático completo
Claude crea y cancela citas directamente en Medilink. Recordatorios automáticos. Manejo de errores. Pruebas con pacientes reales.

🟡 Dashboard de monitoreo
Panel para el equipo: ver conversaciones, métricas, intervención manual. Mejoras continuas según feedback.
