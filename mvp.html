🔀 Opciones A / B / C — ¿Cuál elegir?
Opción A — Backend + LLM libre
Claude maneja toda la conversación con herramientas. Flujo completamente natural, sin menús.
  • Experiencia más natural
  • Un solo prompt lo controla todo
  • Fácil de extender
  • Más tokens = más costo
  • Riesgo de alucinaciones en fechas/horas
  • Más difícil de depurar
Opción B — Flujo guiado sin LLM
Menú numerado: 1. Agendar / 2. Cancelar / 3. Ver horarios / 4. Hablar con recepción.
  • Costo prácticamente $0 en LLM
  • Sin errores de interpretación
  • Más fácil de implementar
  • Experiencia rígida y fría
  • No entiende texto libre
  • Pacientes frustrados al salirse del flujo
⭐ RECOMENDADA
Opción C — Mixta (LLM + flujo controlado)
Claude detecta la intención y extrae datos. Luego el sistema toma el control con pasos guiados antes de ejecutar.
  • Natural al escribir, seguro al ejecutar
  • LLM solo en entrada → bajo costo
  • Confirmación explícita antes de crear cita
  • Fallback a humano cuando Claude no entiende
  • Mejor relación costo/beneficio
  • Algo más de código (dos capas)
  • Estado de sesión necesario

📋 Pasos concretos para implementar
1
🟢 Crear cuenta Twilio + conectar WhatsApp Sandbox 30 min
Twilio ofrece un sandbox de WhatsApp que funciona inmediatamente sin verificación Meta. Perfecto para desarrollar y probar con pacientes piloto.
  • Crear cuenta en twilio.com (gratis)
  • Ir a Messaging → Try it out → Send a WhatsApp message
  • Escanear QR con tu WhatsApp para unirte al sandbox
  • Anotar: Account SID, Auth Token, número sandbox (+14155238886)
  • Instalar: pip install twilio
2
🔵 Levantar servidor FastAPI con endpoint /webhook 1 hora
FastAPI recibe el POST de Twilio con el mensaje del paciente y responde. Necesita URL pública HTTPS — usar ngrok en desarrollo.
  • Instalar: pip install fastapi uvicorn python-dotenv
  • Crear main.py con endpoint POST /webhook
  • Instalar ngrok: ngrok http 8000
  • Copiar URL pública de ngrok → pegar en Twilio Sandbox → "When a message comes in"
  • Probar: enviar "hola" desde WhatsApp y ver el log del servidor
3
🟣 Crear wrapper de Medilink API 2–4 horas
Encapsular los endpoints de Medilink en 5 funciones Python simples. Esto desacopla el bot de los detalles de la API y facilita testear.
  • Crear medilink.py con las 5 funciones básicas
  • Usar httpx para llamadas async: pip install httpx
  • Guardar credenciales en .env (MEDILINK_API_URL, MEDILINK_TOKEN)
  • Crear medilink_mock.py con datos de prueba para desarrollo
  • Test unitario: llamar cada función y verificar respuesta
4
🟠 Integrar Claude para detección de intención 2–3 horas
Claude recibe el mensaje y devuelve JSON estructurado con intención y datos extraídos. No responde al usuario directamente — solo clasifica.
  • Instalar: pip install anthropic
  • Crear claude_helper.py con función detect_intent(mensaje, historial)
  • Prompt devuelve JSON: {"intent": "agendar", "especialidad": "kinesiología", ...}
  • Usar claude-haiku-4-5 (más barato, suficiente para clasificación)
  • Probar con 20 mensajes de prueba típicos
5
🟡 Implementar máquina de estados para flujos 4–6 horas
Cada conversación tiene un estado: IDLE → ESPERANDO_RUT → MOSTRANDO_SLOTS → CONFIRMANDO → DONE. SQLite guarda el estado por número de WhatsApp.
  • Crear tabla sessions: (phone, state, data_json, updated_at)
  • Implementar handle_message(phone, texto) con switch por estado
  • IDLE: llamar a Claude → detectar intención → cambiar estado
  • ESPERANDO_RUT: validar RUT → buscar_paciente() → mostrar slots
  • MOSTRANDO_SLOTS: usuario elige número → estado CONFIRMANDO
  • CONFIRMANDO: usuario dice "sí" → crear_cita() → DONE
  • Timeout de sesión: si no responde en 30 min → volver a IDLE
6
🟢 Deploy y prueba con pacientes piloto 1–2 días
Deploy en VPS con Docker. Prueba con 5–10 pacientes conocidos del centro antes de lanzar públicamente.
  • Dockerizar: Dockerfile + docker-compose.yml (FastAPI + SQLite o PostgreSQL)
  • Deploy en VPS con dominio propio (HTTPS con Let's Encrypt)
  • Actualizar Twilio webhook a URL de producción
  • Prueba real: 5 pacientes conocidos agendan y cancelan
  • Monitorear logs manualmente al inicio
  • Cuando estable: iniciar verificación Meta para número propio del CMC

⚡ main.py — Webhook principal
from fastapi import FastAPI, Form
from twilio.rest import Client
from session import get_session, save_session
from flows import handle_message

app = FastAPI()
twilio_client = Client(ACCOUNT_SID, AUTH_TOKEN)

@app.post("/webhook")
async def webhook(
    From: str = Form(...),  # número WhatsApp
    Body: str = Form(...),  # texto del paciente
):
    phone = From.replace("whatsapp:", "")
    session = get_session(phone)

        response = await handle_message(phone, Body, session)

    twilio_client.messages.create(
        body=response,
        from_="whatsapp:+14155238886",
        to=From
    )
    return {"ok": True}
🧠 claude_helper.py — Detección de intención
import anthropic, json

client = anthropic.Anthropic()

INTENTS = ["agendar", "cancelar", "ver_reservas",
           "precio", "info", "humano", "otro"]

async def detect_intent(mensaje: str) -> dict:
    resp = client.messages.create(
        model="claude-haiku-4-5-20251001",
        max_tokens=200,
        system="""Eres clasificador para Centro Médico Carampangue.
Devuelve SOLO JSON sin markdown:
{"intent": "...", "especialidad": "...", "fecha": "..."}
Intents válidos: agendar, cancelar, ver_reservas,
precio, info, humano, otro""",
        messages=[{"role": "user", "content": mensaje}]
    )
    return json.loads(resp.content[0].text)
📅 medilink.py — Wrapper API
import httpx
from config import MEDILINK_URL, MEDILINK_TOKEN

headers = {"Authorization": f"Bearer {MEDILINK_TOKEN}"}

async def buscar_disponibilidad(especialidad, fecha=None):
    r = await httpx.AsyncClient().get(
        f"{MEDILINK_URL}/disponibilidad",
        params={"especialidad": especialidad, "fecha": fecha},
        headers=headers
    )
    return r.json()

async def crear_cita(rut, profesional_id, slot_id):
    r = await httpx.AsyncClient().post(
        f"{MEDILINK_URL}/citas",
        json={"rut": rut, "profesional_id": profesional_id,
              "slot_id": slot_id},
        headers=headers
    )
    return r.json()

🔀 flows.py — Máquina de estados
async def handle_message(phone: str, texto: str, session: dict) -> str:
    state = session.get("state", "IDLE")

    # ── Estado IDLE: detectar intención ─────────────────────────────
    if state == "IDLE":
        intent_data = await detect_intent(texto)
        intent = intent_data.get("intent")

        if intent == "agendar":
            session.update({"state": "WAIT_RUT", "accion": "agendar",
                            "especialidad": intent_data.get("especialidad")})
            save_session(phone, session)
            return "¡Hola! Para agendar necesito tu RUT. ¿Cuál es?"

        elif intent == "cancelar":
            session.update({"state": "WAIT_RUT", "accion": "cancelar"})
            save_session(phone, session)
            return "Claro. Para cancelar necesito tu RUT. ¿Cuál es?"

        elif intent in ("precio", "info"):
            return await claude_faq(texto)  # Claude responde directo

        else:
            return "Hola 👋 Puedo ayudarte a:\n1. Agendar una hora\n2. Cancelar una hora\n3. Ver tus reservas\n4. Hablar con recepción\n\n¿Qué necesitas?"

    # ── Estado WAIT_RUT ──────────────────────────────────────────────
    elif state == "WAIT_RUT":
        rut = clean_rut(texto)
        if not valid_rut(rut):
            return "RUT inválido. Ingresa tu RUT (ej: 12.345.678-9)"
        paciente = await buscar_paciente(rut)
        if session["accion"] == "agendar":
            slots = await buscar_disponibilidad(session["especialidad"])
            session.update({"state": "SHOW_SLOTS", "rut": rut, "slots": slots})
            save_session(phone, session)
            return format_slots(slots)  # "1. Mar 25 · 09:00 ..."

    # ── Estado CONFIRMING: crear cita ────────────────────────────────
    elif state == "CONFIRMING":
        if texto.strip().lower() in ("sí", "si", "yes", "ok"):
            resultado = await crear_cita(session["rut"], session["slot"])
            session["state"] = "IDLE"
            save_session(phone, session)
            return f"✅ ¡Hora reservada! {resultado['confirmacion']}"
