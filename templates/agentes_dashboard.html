01 — Equipo de auditores · read-only

Los que revisan.

Revisan código, logs y producción desde un ángulo específico. Nunca escriben — solo levantan hallazgos para decidir qué arreglar.

Auditor
cmc-bugs
Cazador general de bugs
Pasada completa por logs recientes y código en busca de bugs sin foco específico. Útil tras deploy o como ronda regular.
Bash · Read · Glob · Grep
Auditor
cmc-medical
Exactitud médica & clínica
Detecta precios desactualizados, derivaciones erróneas, triage que pierde patología grave o info de profesionales desfasada.
Bash · Read · Glob · Grep
Auditor
cmc-performance
Performance & escalabilidad
Queries lentas, N+1, cache stampede, memory leaks, locks demasiado amplios, httpx clients no reutilizados.
Bash · Read · Glob · Grep
Auditor
cmc-security
Seguridad & compliance
Leaks de tokens, SQL injection, CSRF, rate-limiting, validación. Compliance Ley 21.719 y WhatsApp Business Policy.
Bash · Read · Glob · Grep
Auditor
cmc-ux
UX conversacional
Mensajes confusos, flujos rotos, botones mal etiquetados, loops, redundancias y problemas de tono/formato del bot.
Bash · Read · Glob · Grep
Auditor
cmc-conversation-auditor
Conversaciones reales en prod
Trabaja directo sobre sessions.db: detecta bugs en producción, casos edge, frustración del paciente, oportunidades comerciales perdidas.
Bash · Read · Glob · Grep · SQLCipher

07 — Cómo invocarlos

Tres formas.

01 · AUTOMÁTICO

Por descripción

Le pides algo a Claude principal. Si la tarea calza con la description de un agente, lo invoca solo.

"analiza la conversión por especialidad"
→ cmc-data-analyst

02 · EXPLÍCITO

Por nombre

Le indicas qué agente usar. Claude obedece sin interpretar.

"usá el agente cmc-bot-engineer para arreglar el bug en flows.py"

03 · INTERNO · TASK

Por API

Internamente Claude usa la herramienta Agent.

Agent(subagent_type=
"cmc-medical",
prompt="...")
